# Lab8 构建超级英雄列表应用 实验报告

---

## 一、应用整体结构说明
项目采用分层模块化结构，职责拆分清晰：
1.  model 层：存放 Hero 数据类、HeroesRepository 静态数据源，统一管理实体数据与模拟数据。
2.  ui/theme 层：自定义 Material3 颜色、形状、字体排版、全局主题，统一控制全局 UI 样式。
3.  页面层
    - HeroesScreen.kt：封装英雄列表 LazyColumn 和单个列表项 HeroItem 可组合函数。
    - MainActivity.kt：程序入口，通过 Scaffold 整合顶部应用栏与列表主体，承载全局主题。
4.  资源层：strings 字符串资源、drawable 英雄图片资源、font 自定义 Cabin 字体资源。

整体遵循**数据与 UI 分离、样式统一主题管理、组件复用**的 Compose 开发思想。

---

## 二、Hero 数据类设计与理由
### 1. 字段说明
- `@StringRes val nameRes: Int`：英雄名称字符串资源 ID
- `@StringRes val descriptionRes: Int`：英雄描述字符串资源 ID
- `@DrawableRes val imageRes: Int`：英雄头像图片资源 ID

### 2. 设计理由
1.  使用数据类 `data class` 自动提供解构、equals、toString 等方法，适合列表实体模型。
2.  加入 `@StringRes`、`@DrawableRes` 资源注解，编译时校验资源 ID 类型，避免传错资源。
3.  不直接存放字符串和图片位图，只存资源 ID，符合 Android 资源规范化管理，支持多语言、主题切换。

---

## 三、HeroesRepository 数据源组织方式
1.  采用 `object` 单例对象作为仓库类，全局唯一、无需实例化，适合静态模拟数据源。
2.  内部定义 `heroes` 只读 List，一次性初始化 6 个 Hero 对象，绑定对应字符串和图片资源。
3.  数据集中托管在 Repository，UI 层只需直接调用 `HeroesRepository.heroes`，实现数据与视图解耦。
4.  后续如需改为网络请求、本地数据库，只需修改 Repository 内部实现，UI 层无需改动。

---

## 四、英雄列表项布局实现思路
1.  外层使用 Material3 `Card` 作为卡片容器，采用主题中等圆角、轻微阴影，符合设计规格。
2.  卡片内部采用 `Row` 水平布局：左侧 `Column` 文字、右侧正方形头像。
3.  文字区域展示英雄名称与简介，统一使用主题字体样式，保证视觉一致性。
4.  头像设置固定尺寸并做圆角裁剪，适配卡片整体风格，布局比例协调美观。
5.  通过修饰符设置卡片内外边距，保证列表项之间留白合理，界面整洁不拥挤。

---

## 五、LazyColumn 列表实现和间距配置说明
1.  使用 `LazyColumn` 实现长列表懒加载，仅渲染屏幕可见条目，相比普通 Column 性能更优。
2.  通过 `contentPadding` 设置列表整体上下内边距，防止内容紧贴屏幕边缘。
3.  利用 `verticalArrangement.spacedBy` 统一设置列表项垂直间距，无需在每个条目重复设置边距。
4.  调用 `items` 方法遍历英雄数据源，自动循环生成每一个 `HeroItem` 列表组件。
5.  列表容器与列表项单独封装，结构解耦，便于后期修改样式和扩展功能。

---

## 六、Material 主题配置说明，包括颜色、字体和形状
1.  颜色配置：定义浅色、深色两套配色方案，设置主色调、文字前景色、背景色等全局参数，适配系统明暗模式切换。
2.  字体配置：引入自定义 Cabin 字体，统一配置标题、正文文字字号与字重，全局文本自动跟随主题字体样式。
3.  形状配置：预设小、中、大三种圆角规格，分别应用于图片、卡片、弹窗等组件，保持全局圆角风格统一。
4.  通过自定义主题函数整合颜色、字体、形状配置，整个应用所有组件默认继承主题属性，统一 UI 设计规范。

---

## 七、顶部应用栏和状态栏处理说明
1.  借助 `Scaffold` 脚手架组件内嵌 `TopAppBar`，快速搭建页面顶部导航栏结构。
2.  应用栏标题引用字符串资源，配色关联主题主色，适配明暗主题自动变色。
3.  利用 Scaffold 自带 `innerPadding` 避让顶部应用栏，避免列表内容被标题栏遮挡。
4.  适配状态栏沉浸式效果，让状态栏颜色与顶部应用栏保持一致，页面整体视觉更连贯。
5.  应用栏样式统一遵循 Material3 设计规范，简洁大方且适配不同尺寸设备屏幕。

---

## 八、遇到的问题与解决过程
1.  问题：列表项间距大小不一，部分卡片出现重叠或留白过大。
    解决：取消单个卡片冗余边距，统一在 `LazyColumn` 中配置全局条目间距，规范整体留白。
2.  问题：图片显示拉伸变形，圆角效果失效。
    解决：固定图片尺寸，设置内容居中裁剪，绑定主题形状做圆角裁剪，恢复正常展示样式。
3.  问题：深色模式下顶部标题文字对比度不足，看不清内容。
    解决：应用栏文字颜色跟随主题 `onPrimary` 自动适配，明暗模式下都能保证清晰可读。
4.  问题：硬编码字符串无法适配多语言切换。
    解决：全部改用资源 ID 引用，数据类只存资源索引，通过资源方法动态加载文字内容。
5.  问题：页面内容被状态栏、顶部应用栏遮挡。
    解决：接收 Scaffold 内部边距参数，给列表外层添加内边距避让系统栏与导航栏区域。
