## 一、应用功能与展示信息
本实验基于 Jetpack Compose 框架开发课程主题展示应用，核心功能与展示信息如下：
- 以**两列垂直网格**形式展示多类课程主题卡片，包含建筑、生物、设计、音乐、法律等二十余种主题信息

- 界面支持**垂直滚动**，可流畅查看所有主题内容，适配不同屏幕尺寸
- 严格按照设计稿实现边距、尺寸、间距规范，界面布局整齐、美观、统一
- 采用声明式 UI 构建方式，代码简洁易读，支持 Android Studio 实时预览
- 实现数据与界面分离，便于后续扩展主题内容与修改界面样式

## 二、布局结构说明
本次界面采用**模块化、分层嵌套**的结构设计，各组件职责清晰，布局逻辑明确：
1. **根布局**：使用 `Scaffold` 作为页面根容器，填充整个屏幕空间，适配系统状态栏与导航栏，提供稳定的页面框架
2. **网格布局**：使用 `LazyVerticalGrid` 实现核心可滚动两列网格，配置固定列数、卡片间距与整体内边距
3. **卡片容器**：使用 `Card` 组件实现卡片样式，具备默认圆角与阴影效果，提升界面视觉层次
4. **卡片内部**：采用 `Row` 水平布局，将图片区域与文字区域左右排列
   - 左侧图片区域：固定尺寸方形图片，按比例裁剪显示，保证图片不变形
   - 右侧文字区域：使用 `Column` 垂直布局，依次展示主题名称与课程数量
   - 底部信息行：使用 `Row` 水平布局，将图标与课程数字对齐展示，提升美观度

整体布局遵循 Compose 最佳实践，组件复用性强，结构易于维护。

## 三、关键技术实现
1. **可滚动网格实现**
使用 `LazyVerticalGrid` 懒加载网格组件，搭配 `GridCells.Fixed(2)` 实现固定两列布局，自带垂直滚动能力，仅加载屏幕可见区域卡片，提升应用运行性能。

2. **界面样式与边距控制**
通过 `Modifier` 修饰符精准控制界面样式，严格匹配设计规范：
- 统一设置卡片间距、整体内边距为 8dp
- 图片固定尺寸 68dp×68dp，保证所有图片大小一致
- 文字区域设置标准内边距，标题与课程信息保持合理间距
- 图标与文字对齐显示，界面风格统一规范
- 把图片封装在一个数据源中，减少了代码中的复杂性，直接把图片的包导过去代码中，使用起来更便捷

3. **数据模型与数据源**
定义 `Topic` 数据类封装主题信息，包含主题名称、课程数量、图片资源；通过 `DataSource` 单例对象统一管理所有主题数据，实现数据与 UI 完全解耦，便于维护与扩展。

4. **资源引用与组件使用**
使用字符串资源、图片资源、图标资源统一管理界面内容；组合使用 `Image`、`Text`、`Icon`、`Row`、`Column` 等基础组件，快速构建标准化界面。
